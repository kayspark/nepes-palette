# Nepes Chart Palette ‚Äî Design Spec

## Overview

Create chart color palettes for R (ggplot2) and Python (matplotlib) derived from the nepes corporate colorscheme. Light theme is default. Two new repos: `ggnepes` (R/CRAN) and `mplstyle-nepes` (Python).

Includes core palette adjustments (green, purple, chart ordering) based on Gemini 2.5 Pro colorblind/perceptual analysis.

## Pre-requisite: Core Palette Updates

Before building ggnepes, update `palette_data.py`:

### Color adjustments (both light and dark)
- **Green (light):** `#2A8030` ‚Üí `#4EAF56` (lighter, colorblind-safe vs red)
- **Purple (light):** `#7A4FA0` ‚Üí `#9A429A` (hue-shifted away from blue)
- Dark theme equivalents to be computed for perceptual consistency

### Chart base reorder
Old: blue, orange, green, magenta, teal, red
New: **blue, orange, green, red, teal, purple**

Rationale (Gemini): maximizes perceptual distance between adjacent colors, blue-orange is colorblind-safe 1-2 pair for dual-axis charts.

### 12-color paired strategy
Replace `chart_series()` lightening with explicit dark/light pairs per hue:

**Light theme (on white bg):**

| # | Name   | Dark variant | Light variant |
|---|--------|-------------|---------------|
| 1 | Blue   | `#23438E`   | `#8A9BC4`     |
| 2 | Orange | `#D08A10`   | `#F5C86E`     |
| 3 | Green  | `#4EAF56`   | `#A9D8AC`     |
| 4 | Red    | `#C4181F`   | `#EAA8AB`     |
| 5 | Teal   | `#2D7A82`   | `#8EC2C6`     |
| 6 | Purple | `#9A429A`   | `#CA9ACA`     |

Default cycle: 6 dark variants. Full 12: interleaved dark/light pairs.

Dark theme pairs to be computed from dark palette bases.

### SPC semantic palette
Added to core palette (not chart-only):

| Role | Light hex | Dark hex | Usage |
|------|-----------|----------|-------|
| Center line (CL) | `#23438E` | `#5C8CFF` | X-bar mean, EWMA center |
| Data points | `#B0C4DE` | `#6B8AD8` | Normal observations |
| Control limits (UCL/LCL) | `#7F7F7F` | `#8A9199` | Statistical bounds |
| Spec limits (USL/LSL) | `#D08A10` | `#FEA413` | Engineering specs |
| Violations | `#C4181F` | `#FF5C5C` | Out-of-control points |

**API:**

```r
library(ggnepes)

# Palette access
nepes_pal("light")           # 12-color vector (6 dark + 6 light pairs)
nepes_pal("light", n = 6)    # first 6 (dark variants only)
nepes_pal("dark")            # dark theme variant
nepes_pal()                  # default = "light"

# SPC palette
nepes_spc("light")           # named vector: cl, data, ucl, usl, violation
nepes_spc()                  # default = "light"

# ggplot2 scales (discrete)
scale_color_nepes(theme = "light")   # default
scale_fill_nepes(theme = "dark")

# ggplot2 themes (optional)
theme_nepes_light()    # white bg, styled axes/grid/text
theme_nepes_dark()     # #1E1C1A bg, light text
```

**Theme functions:**

`theme_nepes_light()`:
- White background, light gray grid lines
- Font: system sans-serif (no custom font dependency)
- Axis text/title colors from light palette fg ramp
- No border, clean minimal style

`theme_nepes_dark()`:
- Background `#1E1C1A`, panel `#1E1C1A`
- Grid lines `#3E3A38`, text `#DCD8D4`
- Same minimal structure

**CRAN requirements:**
- No compiled code (pure R)
- roxygen2 documentation for all exported functions
- testthat tests
- Examples in man pages
- DESCRIPTION: Title, Description, Authors@R, License, Imports

### 3. Generators in `nepes-palette`

Two new generators:
- `generators/ggplot2.py` ‚Äî produces `R/palettes.R` with hex vectors + SPC palette
- `generators/mplstyle.py` ‚Äî produces `.mplstyle` files with paired 12-color cycle

Registered in `generators/__init__.py` alongside existing 16 generators.

## Generation Flow

```
palette_data.py  ‚Üí  generators/ggplot2.py   ‚Üí  ggnepes/R/palettes.R
                 ‚Üí  generators/mplstyle.py  ‚Üí  mplstyle-nepes/nepes-{dark,light}.mplstyle
```

The R package and mplstyle files contain hardcoded hex values (no runtime dependency on Python). Regenerate when palette changes.

- Font bundling
