#+TITLE: Nepes Corporate Colorscheme
#+AUTHOR: kaypark
#+DESCRIPTION: Single source of truth palette with validation and generation tools

* Overview

Nepes-palette is the canonical color definition for the Nepes corporate
colorscheme suite. All tool-specific themes derive from a single file:
=palette_data.py=.

The palette uses a two-tier system:

1. *Named colors* -- brand colors, shade ramps (6-step background, 4-step
   foreground), core and extended hues with dim/bright variants.
2. *Semantic mappings* -- role-based assignments (=keyword=, =function=,
   =string=, =type=, etc.) that map abstract roles to named colors.

Both *dark* and *light* variants are defined, each with full ANSI 16 color
sets. All foreground-on-background pairs target WCAG AA compliance (contrast
ratio >= 4.5:1).

* CLI Usage

The CLI validates the palette, renders a visual swatch, and generates
configs for terminal tools.

** Validate palette

Check WCAG contrast ratios and color harmony:

#+begin_src sh
uv run python -m nepes_palette.cli validate
#+end_src

** Generate swatch image

Render a visual palette sheet to PNG:

#+begin_src sh
uv run python -m nepes_palette.cli swatch -o swatch.png
#+end_src

** Generate tool configs

Generate all tool-specific theme files into a target directory:

#+begin_src sh
uv run python -m nepes_palette.cli generate -o ..
#+end_src

* Generated Themes

The =generate= command produces configs for 18 tools:

#+begin_src sh
uv run python -m nepes_palette.cli generate -o ..
#+end_src

Run =validate= before generating to ensure WCAG AA compliance.

* Installation (per tool)

** bat

Copy the tmTheme files and rebuild the cache:

#+begin_src sh
mkdir -p ~/.config/bat/themes
cp nepes-dark.tmTheme nepes-light.tmTheme ~/.config/bat/themes/
bat cache --build
#+end_src

Set the theme in =~/.config/bat/config=:

#+begin_src conf
--theme="nepes-dark"
#+end_src

** delta

Delta reuses bat's syntax themes. Install the bat theme first (see above),
then add to your =~/.gitconfig=:

#+begin_src gitconfig
[include]
    path = /path/to/nepes.gitconfig
#+end_src

Or copy the relevant color values into your existing delta config and set:

#+begin_src gitconfig
[delta]
    syntax-theme = nepes-dark
#+end_src

** lazygit

Copy the theme values into =~/.config/lazygit/config.yml= under the
=gui.theme= section.

** fzf

Source the generated script in your shell rc file:

#+begin_src sh
# ~/.bashrc or ~/.zshrc
source /path/to/nepes-dark.sh
#+end_src

Or copy the =export FZF_DEFAULT_OPTS= line directly into your rc file.

** lsd

Copy the generated YAML to the lsd config directory:

#+begin_src sh
cp nepes-dark.yaml ~/.config/lsd/colors.yaml
#+end_src

Set the theme in =~/.config/lsd/config.yaml=:

#+begin_src yaml
color:
  theme: custom
#+end_src

** fish

Copy the generated fish script to the auto-loaded config directory:

#+begin_src sh
cp nepes-dark.fish ~/.config/fish/conf.d/nepes-colors.fish
#+end_src

Fish sources all files in =conf.d/= automatically on startup.

** Prerequisites

- Python 3.11+

** Running tests

#+begin_src sh
uv run pytest tests/ -v
#+end_src

** Project structure

#+begin_example
nepes-palette/
  pyproject.toml                   # project metadata (hatchling build)
  src/nepes_palette/
    palette_data.py                # single source of truth (Python dict)
    colors.py                      # computed helpers: rgba, lighten, darken
    cli.py                         # validate / swatch / generate / capture
    validators.py                  # WCAG contrast and harmony checks
    generators/                    # 18 per-tool config generators
    capture/                       # VHS tape builders and runner
  tests/                           # 79 tests
#+end_example
